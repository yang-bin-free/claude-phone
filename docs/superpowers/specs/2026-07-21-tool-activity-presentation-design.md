# CodeAfar 工具活动分层展示设计

**日期：** 2026-07-21  
**状态：** 已确认

## 目标

CodeAfar 不再把模型调用内部工具时产生的名称、参数或原始命令当作普通聊天消息展示。用户应优先看到任务进度、需要自己决定的授权请求和最终结果，而不是 `Bash`、`Read`、`/bin/bash -lc` 等实现细节。

## 展示规则

### 普通聊天与历史记录

- 只保留用户消息、助手回复、排队状态、错误和最终结果。
- 忽略实时及历史中的 `tool_use` 事件，不生成工具消息气泡。
- 工具事件不得打断或拆分助手正在流式生成的正文。
- 旧会话中已经保存的工具事件也按同一规则隐藏。

### 运行状态

- 本次改动不新增持久化的工具状态消息。
- 现有连接、排队和健康状态继续使用独立状态区域，不混入聊天正文。
- 后续若增加“正在检查项目”等进度，应作为临时状态呈现，并在本轮完成后消失。

### 授权与安全

- 隐藏工具活动不改变 Claude 或 Codex 的权限模式、沙箱和执行行为。
- 必须由用户批准的高风险操作仍通过专用授权界面展示；授权界面可以提供可展开的具体命令和影响范围。
- 本次改动不改变服务端协议与历史存储，保留未来诊断、审计和授权界面使用完整工具数据的能力。

### 管理与诊断

- 普通聊天不承担诊断职责。
- 完整命令、工具参数和输出只允许在明确标识的诊断界面中按需查看；本次改动不新增该诊断视图。

## 实现范围

- Mac 和 Android 共用的 Web 聊天客户端停止渲染实时及历史 `tool_use` 事件。
- iOS 原生聊天客户端采用相同行为。
- 删除仅用于普通聊天工具卡片的格式化资源，避免以后误接回原始命令展示。
- 服务端继续翻译、传输和保存工具事件，确保执行、恢复和审计能力不受影响。

## 验收标准

1. Claude 或 Codex 执行 shell 命令时，普通聊天中不出现“执行命令”、`Bash` 或命令文本。
2. 读取、搜索或编辑文件时，普通聊天中不出现 `Read {}` 等工具消息。
3. 重新打开包含旧工具事件的会话，聊天历史不显示这些事件。
4. 工具调用前后的助手文本正常连续显示。
5. Claude/Codex 执行、权限模式、队列、错误和最终回复保持可用。
6. Web、Go、iOS 项目验证及 Mac 应用验收全部通过。

## 非目标

- 不删除底层工具事件或审计数据。
- 不改变模型供应商的执行协议。
- 不在本次改动中设计完整的诊断日志浏览器。
